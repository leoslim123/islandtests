ИНФОРМАЦИЯ ПО ЗАЩИТЕ РАБОТЫ - ТЕСТИРОВАНИЕ ISLAND
===================================================

ЧТО СДЕЛАНО:
- Покрыл проект "Остров" тестами на JUnit 5
- 35 тестов: 33 юнит-тестовых + 2 функциональных
- Все тесты проходят (mvn test -> BUILD SUCCESS)
- Написал README с описанием тестов и скриншотом

ТЕСТОВЫЕ КЛАССЫ:
1. SimulationConfigTest (6 тестов) - проверка конфигурации
2. AnimalTest (12 тестов) - логика животных
3. LocationTest (10 тестов) - ячейка острова
4. IslandTest (5 тестов) - создание и заселение острова
5. FunctionalTest (2 теста) - полный цикл жизни + хищник-жертва

КАК ЗАПУСТИТЬ:
  mvn test
  (Maven должен быть установлен, Java 17)

ВОЗМОЖНЫЕ ВОПРОСЫ ПРЕПОДАВАТЕЛЯ И ОТВЕТЫ:

В: Какие техники тестирования использовались?
О: Эквивалентное разбиение - тестировал разные категории животных
   (хищник/травоядное/всеядное). Граничные значения - рост растений до
   максимума, голодание до смерти, удаление при нуле. Негативное
   тестирование - неизвестный вид, размножение без пары.

В: Чем отличаются юнит-тесты от функциональных?
О: Юнит-тест проверяет один метод или класс изолированно. Функциональный
   тест проверяет сценарий целиком - создание острова, заселение, полный
   такт симуляции, проверка что экосистема работает.

В: Почему выбрал именно эти классы для тестирования?
О: SimulationConfig - ядро конфигурации, от него зависят все животные.
   Location - основная единица хранения, потокобезопасная коллекция.
   Animal - базовая логика поведения. Island - модель мира.

В: Что проверяет testFullLifecycle?
О: Создает остров по размерам из конфига, заселяет его, прогоняет один
   полный такт (еда-размножение-движение-голодание-очистка мертвых) и
   проверяет что популяция выжила.

В: Что проверяет testPredatorPreyEcosystem?
О: Помещает волка и 10 кроликов в одну клетку. Волк охотится, проверяю
   что часть кроликов погибла, волк выжил, мертвые удалены.

В: Почему не использовал моки?
О: Классы хорошо связаны через наследование, проще тестировать через
   реальные объекты. Мок нужен когда зависимость тяжелая (база данных,
   сеть), а тут все in-memory.

В: Как обеспечена потокобезопасность в тестах?
О: Юнит-тесты однопоточные, потокобезопасность проверяется косвенно -
   через CopyOnWriteArrayList в Location и AtomicBoolean/AtomicInteger.
   Функциональные тесты работают в одном потоке, но используют те же
   потокобезопасные структуры что и продакшн-код.

В: Почему Island создается с размерами из SimulationConfig?
О: Метод move() у Animal проверяет границы через SimulationConfig.ISLAND_WIDTH
   и ISLAND_HEIGHT, поэтому остров должен соответствовать этим размерам,
   иначе ArrayIndexOutOfBoundsException при перемещении.
